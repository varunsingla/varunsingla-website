2. The shared context layer is the moat -- not the model.

The reason these will land at JP Morgan is the M365 add-in carrying state across Excel/PowerPoint/Word, not the Opus 4.7 weights. For your stack, ask: what's the state object that survives across tools? That's where the value

2. The ten agent templates -- what each one actually does

Each agent ships in three forms: (a) a Cowork plugin (chat-driven, human-in-the-loop), (b) a Claude Code plugin (developer harness, scriptable, version-controlled), and (c) a Claude Managed Agent cookbook (autonomous, multi-hour, full audit trail). Same logic, three deployment surfaces -- pick by autonomy level needed.

AgentWhat it does
Pitch builderPulls comps from S&P; Capital IQ + PitchBook + Moody's, drafts the deck in PowerPoint via the M365 add-in, threads context to the Excel comps model. Replaces the analyst all-nighter.
Meeting preparerReads the calendar, pulls last 6 months of email threads + CRM notes + recent earnings, drafts a one-pager brief with talking points, open questions, and competitive context.
Earnings reviewerReads transcripts + 10-Qs/10-Ks, updates the Excel model in-place, flags thesis-relevant changes (margin compression, guidance shift, capex pivot) against analyst notes.
Model builderBuilds DCF / LBO / merger models from a structured prompt; cites every assumption back to source filings; produces sensitivity tables and a written rationale.
Market researcherFans out across IBISWorld + Third Bridge + GLG/Guidepoint expert transcripts + Verisk + Fiscal AI; returns a cited, structured market map.
Valuation reviewerAudits an existing model -- checks formula consistency, traces dependencies, flags circular refs and hard-coded overrides, scores defensibility.
